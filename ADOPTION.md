## How to Register Your Team

To add your team to this registry:

1. Edit this file (`ADOPTION.md`)
2. Add your team information to the "Using Teams" section
3. Update the statistics and timeline
4. Submit a PR with your changes

### Privacy Options

Teams can choose their level of visibility:

- **Public**: Team name, repository names, and feedback visible
- **Anonymous**: Team identified as "Team X" with repository names omitted
- **Private**: Data used for statistics only, not individually listed

## Adoption Process

### Getting Started

1. **Review Available Actions**: Check the `examples/` directory for workflow templates
2. **Choose Actions**: Select actions that fit your team's workflow
3. **Test in Staging**: Start with non-critical repositories
4. **Roll Out**: Gradually expand to more repositories
5. **Provide Feedback**: Share your experience to help improve the actions

### Support Channels

- **Documentation**: See `README.md` and `AGENTS.md`
- **Examples**: Check `examples/` directory
- **Issues**: Report bugs or request features via GitHub Issues
