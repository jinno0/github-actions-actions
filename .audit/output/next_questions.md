## 仮定の修正方法

認識が異なる場合は、以下の手順で `config/intent.yml` を修正してください：

1. `.audit/config/intent.yml` を開く
2. 該当する仮定（ASM-XXX）を削除または修正
3. 新しい値を `mission` または `quality_attributes` に直接記載
4. 再監査を実行

### 例: ASM-002（目標カバレッジ）を修正する場合

**修正前**:
```yaml
assumptions:
  - id: "ASM-002"
    field: "quality_attributes.test_coverage"
    value: ">= 70%"
    reason: "README.mdで「テストカバレッジ >= 70% を目標」と明記されているため"
    confidence: "high"
```

**修正後**（目標を80%に変更する場合）:
```yaml
# assumptions は削除

quality_attributes:
  - id: "QA-001"
    name: "テストカバレッジ"
    target: ">= 80%"  # ここを修正
    verification:

```
