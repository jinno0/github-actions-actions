# PR-004: Example Files YAML Validation and Fixes

## Summary
全13ActionのexampleファイルについてYAML構文検証を行い、エラーを修正する。

## Motivation
- 現在、exampleファイルのYAML構文が正しいか検証されていない
- ユーザーがコピーして使う雛形として、構文が正確であることが必須
- GitHub Actionsのworkflowファイルとして有効であることを保証する必要がある

## Proposed Changes

### 1. YAML構文検証の実行
```bash
find examples/ -name '*.yml' -exec yamllint {} \;
```

### 2. 検出されたエラーの修正
- YAML構文エラーの修正
- インデントの修正
- 必須フィールドの欠落を補完

### 3. CIでの自動検証追加
```yaml
# .github/workflows/validate-examples.yml
name: Validate Example Files
on: [push, pull_request]
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Install yamllint
        run: pip install yamllint
      - name: Validate all example files
        run: find examples/ -name '*.yml' -exec yamllint {} \;
```

## Expected Impact
- exampleファイルの品質保証
- ユーザーがコピーして使う際のエラー削減
- リポジトリの信頼性向上

## Success Criteria
- 全exampleファイルがyamllintでエラーなし
- CIで自動検証が実行される
- PR作成時に自動検証が走る

## Estimated Effort
- YAML検証実行: 5分
- エラー修正: 30分〜1時間（エラー数による）
- CI設定追加: 15分
- **合計: 約1時間**

## Dependencies
None

## Risks
- 低リスク: 構文修正のみで既存の動作を変更しない
- 既存のexampleファイルのフォーマットが変更される可能性がある
