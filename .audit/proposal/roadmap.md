## Risk Mitigation

### If Coverage is Below 70%
- Document current state as baseline
- Create improvement plan
- Add tests to critical paths first

### If Acceptance Rate is Low
- Review AI review prompts
- Gather feedback from users
- Iterate on review criteria

### If Adoption is Low
- Review which actions are/aren't used
- Survey users for friction points
- Improve onboarding experience
- Consider pilot programs with friendly teams

### If Telemetry Insufficient
- Fall back to manual registration (Approach B in PR-003)
- Encourage teams to self-register in ADOPTION_REGISTRY.md
