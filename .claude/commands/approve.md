# Approve Command - 統合审批屏幕

承認が必要な操作のための統合UI画面を提供します。

## 使用方法

```bash
/approve [type] [id]
```

## パラメータ

- `type` (必須): 承認タイプ
  - `qa`: QAレビュー承認
  - `deploy`: デプロイ承認
  - `pr`: プルリクエスト承認
  - `code-review`: コードレビュー承認
  - `hotfix`: ホットフィックス承認
  - `release`: リリース承認

- `id` (オプション): 承認対象ID（省略時はリスト表示）

## 実行内容

### 承認フロー

1. `/approve` コマンド実行
2. タイプ指定または承認待ちリスト表示
3. 各タイプの詳細情報を確認
4. 承認/拒否/保留の選択
5. 結果の記録と通知

### 承認タイプ別の情報

承認タイプに応じて、以下の情報を確認します:
- **QA承認**: テスト結果、カバレッジ、クリティカル問題数
- **デプロイ承認**: 環境、バージョン、変更内容、リスクレベル、ロールバック計画
- **PR承認**: タイトル、説明、作成者、レビュアー、変更内容、コンフリクト、ステータスチェック
- **コードレビュー承認**: レビューコメント、修正状況、承認者
- **ホットフィックス承認**: 緊急度、影響範囲、適用計画、ロールバック計画
- **リリース承認**: バージョン、リリースノート、変更内容、テスト結果

### 承認後のアクション

承認タイプに応じた後続処理を実行します:
- `qa`: デプロイパイプライン起動
- `deploy`: デプロイ実行
- `pr`: プルリクエストマージ
- `code-review`: 次のレビューフェーズへ進行
- `hotfix`: ホットフィックス適用
- `release`: リリース公開

## 実行例

```bash
# QA承認
/approve qa qa-2024-001

# デプロイ承認
/approve deploy deploy-123

# 承認待ちリスト表示
/approve
```
